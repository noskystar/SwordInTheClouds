# UI 优化设计文档

## 概述

修复4个功能性UI bug，并改进战斗界面的视觉表现。

## 目标 / 非目标

### 目标
- 修复触控相关 UI 的显示与交互问题（桌面端不误显示、移动端不重叠、交互必触发）
- 修复角色行走动画不可见问题
- 修复标题界面背景缺失（并加入容错回退）
- 战斗 UI 视觉升级：更清晰的层次、可读性、状态反馈（不改核心战斗逻辑）

### 非目标
- 不更换/新增美术资源（战斗 UI 仅用 Graphics/Text/Tween 程序化绘制）
- 不重做输入系统与核心战斗系统（仅修 UI 层和必要的桥接）

## 设计与工程约束（必须遵守）
- 基础分辨率 320×180，整数倍缩放；**UI 布局与尺寸以“逻辑像素”计算，不在布局计算中乘/除 zoom**
- UI 元素（菜单/日志/遮罩/提示等）应固定屏幕空间：`setScrollFactor(0)`，并统一 `depth` 分层（HUD 层 / Overlay 层）
- Graphics 重绘需节制：血条/蓝条/ATB 条 **仅在数值变化时重绘**；闪烁/脉冲效果优先用 tween 改 `alpha/scale`
- 场景重进不泄漏：Battle/Title/Overworld 相关 UI 在 scene `shutdown/destroy` 时必须清理（destroy Graphics/Text、停止 tweens/事件监听）

## 问题根因

### 1. 触屏控件在非触屏设备显示 + 按钮重叠
- `TouchControls` 未检测设备类型，始终创建
- 按钮间距 `btnGap` 计算时除以了 `zoom`（=2），导致间距减半
- 按钮半径 `btnR = H * 0.055` ≈ 10px，间距 ≈ 10px，直径 20px，三者并排必然重叠

#### 修复策略
- 默认不创建/不显示触控 UI；当检测到触屏能力或触摸输入后再启用
- 移除 `btnGap` 与 `zoom` 的耦合：所有按钮布局以逻辑像素计算
- 调整按钮半径与间距的关系，确保不重叠：`btnGap >= 2 * btnR + padding`（padding 取 4~6）

### 2. 虚拟E按钮无反应
- `onInteract` 回调先把 `touchInteractWasDown = false`，再调用 `checkInteractions()`
- `checkInteractions()` 仅在 `touchInteractWasDown === true` 时才执行交互逻辑
- 结果：触摸交互永远不会触发

#### 修复策略
- 按下时设置 `touchInteractWasDown = true` 并触发一次 `checkInteractions()`
- 抬起时设置 `touchInteractWasDown = false`，避免重复触发
- 交互触发应与“按钮状态变更”顺序一致：**先置状态，再检查**

### 3. 角色移动时看不到人
- `player_walk.png` 是 1024×1024 的像素风格 2×2 精灵图（4帧）
- `boot-scene.ts` 加载时设置 `frameWidth=16, frameHeight=16`
- Phaser 切出的是大图左上角 16×16 的空白小块，行走动画播放时看不到角色

#### 风险与决策（必须在实现前完成）
这里存在“资源规格与项目像素约束”潜在不一致：如果该图确实是 2×2（4 帧），单帧尺寸应为 512×512，这与常见像素角色（16/32）不匹配。

实现时采用以下决策规则（不改资源前提下）：
- 以 `player_walk.png` 的真实尺寸为准（运行时可从纹理读取 `source.width/height`），若能整除 2，则按 `frameWidth = w / 2`、`frameHeight = h / 2` 切帧
- 若纹理尺寸不满足该规则（或切帧后仍为空白/不可见），则回退到“以配置/常量指定帧尺寸”的旧逻辑，并在控制台输出明显告警，便于后续修正资源或配置

验收以“动画帧可见”为准（见下文验收标准），不以某个固定帧尺寸为准。

### 4. 标题界面没有背景图
- `bg_title.png`（320×180）文件存在且 BootScene 已加载
- 需要添加纹理存在性检查 + 程序化回退背景，防止加载失败时无图

#### 修复策略
- TitleScene 创建背景前先检查纹理 key 是否存在
- 若缺失：用 Graphics 生成回退背景（深色渐变/纯色 + 简单噪点/网格均可），保证标题场景永不“黑屏”

## 战斗UI改进设计

全部使用 Phaser Graphics + Text + Tween 程序化实现，无需额外图片素材。

### 实体显示区
- 每个实体添加半透明圆角底板（`#1a1a2e`，alpha=0.7，圆角4px，1px灰色边框）
- 精灵居中于底板上方，无精灵时保留彩色矩形加白色描边
- 名称 12px + 金色描边，五行标签 9px 在名称右侧
- 血条/蓝条/ATB条在底板内整齐排列

### 血条/蓝条/ATB条
- 使用 Graphics 绘制圆角条形容器
- 血条：三段式颜色（满血 `#44aa44`、半血 `#aaaa44`、低血 `#aa4444`）+ 水平渐变
- 蓝条：固定蓝色 `#4488cc` + 轻微渐变
- ATB条：黄色 `#ffcc00`，满值时白色脉冲闪光（tween alpha 0.8→1.0）
- 所有条添加 1px 白色半透明边框（alpha=0.2）

### 战斗菜单
- 圆角矩形面板背景（`#0d0d1a`，alpha=0.92，圆角6px，1px `#4a4a6a` 边框）
- 宽度 160px，高度自适应
- 选中项：左侧金色箭头 "▶" + 文字变白 + 背景高亮条（`#2a3a5a`，alpha=0.6）
- 未选中项：文字 `#888888`
- 不可选项：文字 `#444444` + 删除线
- 底部按键提示："↑↓ 选择 · 空格 确认 · ESC 取消"

### 战斗日志
- 半透明背景面板（`#000000`，alpha=0.5，圆角4px，宽度300px）
- 顶部居中，padding 6px，文字左对齐，行高 14px
- 颜色编码：伤害 `#ff6666`（暴击 `#ffcc00`）、治疗 `#66ff66`、系统 `#cccccc`、剑意 `#ffcc00`
- 新消息从底部滑入（y+10→0，alpha 0→1，200ms）

### 剑意显示
- 进度条容器圆角化，宽度 56px
- 未满时：金色 `#ffaa00` 渐变
- 满100时：橙红色 `#ff6600` + 脉冲发光（外发光 `#ff6600`，alpha 0→0.5 循环）
- 满值时文字旁添加 "✦" 图标闪烁

### 回合指示器（新增）
- 当前行动实体上方添加金色闪烁三角形 "▲"，每 600ms 上下浮动 3px
- 玩家回合：屏幕四周边缘淡蓝色遮罩（`#4488ff`，alpha=0.08）
- 敌人回合：屏幕边缘淡红色遮罩（`#ff4444`，alpha=0.05）

### 目标选择指示器
- 箭头改为发光金色（`#ffdd00`，2px外发光）
- 左右摆动动画（x±2px，400ms循环）
- 被选中敌人底板边框金色高亮

## 验收标准（Definition of Done）

### 触屏控件（`TouchControls`）
- 桌面端（无触摸输入）默认不显示触控 UI
- 触屏端显示后，移动/交互按钮点击区域不重叠，连续快速点击不会误触相邻按钮
- 不同整数倍缩放下（例如 zoom=2/3/4）按钮布局一致、无错位（以逻辑像素布局为准）

### 虚拟 E 交互（Overworld）
- 按下虚拟 E：当角色面向可交互目标时，**必定触发一次**交互
- 抬起虚拟 E：不会重复触发；长按不会在同一帧内多次触发

### 角色行走动画可见性
- 行走动画播放时角色主体可见（非空白帧），静止/行走切换无明显闪烁
- 动画帧切割规则正确：切割后帧矩形落在有效像素区域（不会一直取到左上角空白）

### 标题背景
- 正常情况下显示 `bg_title`，并按 320×180 逻辑分辨率对齐/铺放
- 当纹理 key 缺失或加载失败时，显示程序化回退背景（可通过 dev 临时改 key 模拟验证）

### 战斗 UI（视觉与性能）
- 实体底板、条形（HP/MP/ATB）、战斗菜单、战斗日志、剑意显示、回合指示器、目标选择指示器均能正常显示且不遮挡关键内容
- 条形类 UI 仅在数值变化时重绘；脉冲/闪烁效果使用 tween（不做每帧渐变重绘）
- 退出战斗/切场景后不残留 UI（无重复叠加、无事件监听/ tween 泄漏导致的二次进入异常）

## 回滚方案
- 所有新 UI 行为保持“可开关”：若出现大范围 UI 回归，可先回退到旧版绘制/布局逻辑（保留 bugfix 必要修复）
- Title 背景回退为纯色背景（最低保底），确保可用性优先
