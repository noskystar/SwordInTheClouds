# UI 优化设计文档

## 概述

修复4个功能性UI bug，并改进战斗界面的视觉表现。

## 问题根因

### 1. 触屏控件在非触屏设备显示 + 按钮重叠
- `TouchControls` 未检测设备类型，始终创建
- 按钮间距 `btnGap` 计算时除以了 `zoom`（=2），导致间距减半
- 按钮半径 `btnR = H * 0.055` ≈ 10px，间距 ≈ 10px，直径 20px，三者并排必然重叠

### 2. 虚拟E按钮无反应
- `onInteract` 回调先把 `touchInteractWasDown = false`，再调用 `checkInteractions()`
- `checkInteractions()` 仅在 `touchInteractWasDown === true` 时才执行交互逻辑
- 结果：触摸交互永远不会触发

### 3. 角色移动时看不到人
- `player_walk.png` 是 1024×1024 的像素风格 2×2 精灵图（4帧）
- `boot-scene.ts` 加载时设置 `frameWidth=16, frameHeight=16`
- Phaser 切出的是大图左上角 16×16 的空白小块，行走动画播放时看不到角色

### 4. 标题界面没有背景图
- `bg_title.png`（320×180）文件存在且 BootScene 已加载
- 需要添加纹理存在性检查 + 程序化回退背景，防止加载失败时无图

## 战斗UI改进设计

全部使用 Phaser Graphics + Text + Tween 程序化实现，无需额外图片素材。

### 实体显示区
- 每个实体添加半透明圆角底板（`#1a1a2e`，alpha=0.7，圆角4px，1px灰色边框）
- 精灵居中于底板上方，无精灵时保留彩色矩形加白色描边
- 名称 12px + 金色描边，五行标签 9px 在名称右侧
- 血条/蓝条/ATB条在底板内整齐排列

### 血条/蓝条/ATB条
- 使用 Graphics 绘制圆角条形容器
- 血条：三段式颜色（满血 `#44aa44`、半血 `#aaaa44`、低血 `#aa4444`）+ 水平渐变
- 蓝条：固定蓝色 `#4488cc` + 轻微渐变
- ATB条：黄色 `#ffcc00`，满值时白色脉冲闪光（tween alpha 0.8→1.0）
- 所有条添加 1px 白色半透明边框（alpha=0.2）

### 战斗菜单
- 圆角矩形面板背景（`#0d0d1a`，alpha=0.92，圆角6px，1px `#4a4a6a` 边框）
- 宽度 160px，高度自适应
- 选中项：左侧金色箭头 "▶" + 文字变白 + 背景高亮条（`#2a3a5a`，alpha=0.6）
- 未选中项：文字 `#888888`
- 不可选项：文字 `#444444` + 删除线
- 底部按键提示："↑↓ 选择 · 空格 确认 · ESC 取消"

### 战斗日志
- 半透明背景面板（`#000000`，alpha=0.5，圆角4px，宽度300px）
- 顶部居中，padding 6px，文字左对齐，行高 14px
- 颜色编码：伤害 `#ff6666`（暴击 `#ffcc00`）、治疗 `#66ff66`、系统 `#cccccc`、剑意 `#ffcc00`
- 新消息从底部滑入（y+10→0，alpha 0→1，200ms）

### 剑意显示
- 进度条容器圆角化，宽度 56px
- 未满时：金色 `#ffaa00` 渐变
- 满100时：橙红色 `#ff6600` + 脉冲发光（外发光 `#ff6600`，alpha 0→0.5 循环）
- 满值时文字旁添加 "✦" 图标闪烁

### 回合指示器（新增）
- 当前行动实体上方添加金色闪烁三角形 "▲"，每 600ms 上下浮动 3px
- 玩家回合：屏幕四周边缘淡蓝色遮罩（`#4488ff`，alpha=0.08）
- 敌人回合：屏幕边缘淡红色遮罩（`#ff4444`，alpha=0.05）

### 目标选择指示器
- 箭头改为发光金色（`#ffdd00`，2px外发光）
- 左右摆动动画（x±2px，400ms循环）
- 被选中敌人底板边框金色高亮
